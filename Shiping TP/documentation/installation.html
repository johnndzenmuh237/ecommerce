Installation Guide

Get SwiftCargo up and running in minutes. No build tools required — pure HTML, CSS, and JavaScript.

Requirements

SwiftCargo is a static HTML template. There are no server-side requirements, no npm installs, and no build steps needed to get it running.

RequirementMinimumNotes
BrowserChrome 90+, Firefox 88+, Safari 14+, Edge 90+All modern browsers supported
Web ServerAny static file serverApache, Nginx, Netlify, Vercel, GitHub Pages etc.
Internet ConnectionRequired for Google FontsOr self-host fonts (see Customization guide)
File EditorVS Code recommendedAny text editor works

Step-by-Step Setup

1

Extract the Template Files

Unzip the downloaded package into your desired project folder. Keep all folder structures intact — the CSS and JS files reference each other via relative paths.

2

Open in Your Code Editor

We recommend Visual Studio Code with the "Live Server" extension for the best development experience.

# Open in VS Code
code shipping-agency-pro/
3

Add Your Images

Place your images in the appropriate subfolders inside assets/images/. The HTML files reference images by these paths. See the Customization guide for expected image names and dimensions.

ℹ️
Fallback Images All <img> tags include an onerror fallback to a placeholder image from Unsplash, so the template looks great even before you add your own photos.
4

Update Your Brand Information

Open assets/js/main.js and update the SITE configuration object at the top of the file:

const SITE = {
  name:    'YourCompany',       // Appears in header & footer
  tagline: 'Global Logistics',   // Subtitle under logo
  phone:   '+1 (800) 000-0000',
  email:   'info@yourcompany.com',
  address: 'Your Address Here',
  hours:   'Mon–Fri: 9am–5pm EST',
};
5

Launch with Live Server

In VS Code, right-click index.html and select "Open with Live Server". Your browser will open at http://127.0.0.1:5500 with hot-reload on file save.

⚠️
Don't open HTML files directly Opening HTML files via file:// (double-clicking) can cause issues with relative paths and JavaScript modules. Always use a local server.

CSS File Reference

style.css — Core Stylesheet

The main stylesheet containing all design tokens, component styles, layout utilities, and page sections. It is the only required CSS file — the others are enhancements.

Design tokens are defined as CSS custom properties at :root level, making global changes trivial:

/* Change brand color — one line, site-wide */
:root {
  --color-accent: #E8931A;   /* → change to your brand color */
  --color-navy:   #0A1628;
  --font-body:    'Inter', sans-serif;
}

responsive.css — Breakpoints

All media queries are here. Breakpoints used: 1024px (tablet), 768px (mobile), 480px (small mobile). Do not modify style.css breakpoints — add overrides here.

animations.css — Motion

All keyframe animations, scroll-reveal classes (.reveal, .stagger-children), and transition helpers. Includes a prefers-reduced-motion block that disables all animations for accessibility.

themes.css — Color Themes

Pre-built color variants. Apply by adding a class to <html>:

<!-- Available: theme-ocean | theme-forest | theme-crimson | theme-slate | theme-amber -->
<html lang="en" class="theme-ocean">

dark-mode.css — Dark Mode

Dark mode activates automatically via prefers-color-scheme: dark media query, or manually by adding class dark to <html>. The toggle button in the header handles this via JavaScript and persists the preference to localStorage.

JavaScript File Reference

main.js — Core Logic

Handles: header/footer injection, mobile nav toggle, dark mode, scroll reveal (IntersectionObserver), back-to-top button, FAQ accordion, hero tracking form, tab panels, and ripple effects on buttons. Must be included on every page.

tracking.js — Shipment Tracking

Powers the tracking page. Currently uses mock data — replace the MOCK_SHIPMENTS object with real API calls to your carrier's tracking API or TMS system. Include only on tracking.html.

quote-form.js — Multi-Step Form

Manages the 4-step quote request form with client-side validation, step navigation, and a success state. The submitQuoteForm() function at the bottom is where you wire in your backend or email service. Include only on quote-request.html.

counters.js — Animated Numbers

Animates any element with class .counter when it enters the viewport. Configure via data attributes:

<span class="counter"
  data-target="25000"
  data-suffix="+"
  data-duration="2000">0</span>

slider.js — Carousel

A dependency-free slider/carousel. Initializes on any element with data-slider. Supports autoplay, looping, touch/drag, keyboard navigation, and dot indicators:

<div data-slider data-autoplay="4000" data-loop="true">
  <div class="slider-track">
    <div class="slider-slide">…</div>
    <div class="slider-slide">…</div>
  </div>
  <button class="slider-btn slider-btn--prev" data-prev>‹</button>
  <button class="slider-btn slider-btn--next" data-next>›</button>
  <div class="slider-dots" data-dots></div>
</div>

Quick Setup Checklist

✅
Before going live, make sure you have:
  • Updated SITE object in main.js with your real contact info
  • Replaced all placeholder images in assets/images/
  • Updated your company name, logo, and tagline
  • Set your brand color in :root inside style.css
  • Updated the Google Fonts link to use your preferred typefaces (optional)
  • Connected your form submission endpoint in quote-form.js and contact.html
  • Updated all page <title> and <meta name="description"> tags
  • Added your Google Analytics / tracking script (optional)
  • Tested on mobile and in dark mode
