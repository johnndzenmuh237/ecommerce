Customization Guide

Make SwiftCargo your own — update colors, fonts, logo, content, services, and more.

Colors

All colors are defined as CSS custom properties in :root inside assets/css/style.css. Changing them updates the entire site instantly.

To change your primary brand/accent color, update just these three variables:

:root {
  --color-accent:       #E8931A;  /* Main brand color */
  --color-accent-dark:  #C67A10;  /* Hover states */
  --color-accent-light: #F5B04A;  /* Light tints */
}
💡
Pre-built Themes Instead of manually changing colors, try the pre-built themes in themes.css: theme-ocean, theme-forest, theme-crimson, or theme-slate. Apply by adding the class to <html>.

Typography

The template uses three font families, loaded from Google Fonts:

BARLOW CONDENSED — Headings
Inter — Body text, navigation, UI labels (400 / 500 / 600 / 700)
JetBrains Mono — Tracking numbers, code

To change fonts, update both the Google Fonts link in your HTML <head> and the CSS variables:

<!-- In <head> of each HTML file -->
<link href="https://fonts.googleapis.com/css2?family=YOUR+FONT:wght@700;800;900&display=swap" rel="stylesheet">

/* In assets/css/style.css :root */
--font-display: 'Your Font', sans-serif;
--font-body:    'Your Body Font', sans-serif;

Logo & Brand Name

The logo is an inline SVG in main.js (injected into the header and footer). To use your own logo image:

// In assets/js/main.js — injectHeader() function
// Replace the SVG block with:
<img src="assets/images/logo/logo.svg"
     alt="YourCompany"
     class="logo-icon"
     style="height:40px;width:auto;">

Also update the SITE.name and SITE.tagline values in main.js:

const SITE = {
  name:    'YourCompany',
  tagline: 'Your Tagline',
  // ...
};

Icons

The template uses emoji icons throughout (service cards, feature lists, etc.) for zero-dependency simplicity. To use SVG icon libraries instead:

Option A: Heroicons (recommended)

<!-- Add to <head> -->
<script src="https://unpkg.com/@heroicons/react/24/outline"></script>

<!-- Use inline SVG -->
<svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" fill="none" viewBox="0 0 24 24" stroke="currentColor">
  <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M5 13l4 4L19 7" />
</svg>

Option B: Font Awesome

<!-- Add to <head> -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css">

<!-- Usage -->
<i class="fa-solid fa-ship"></i>

Content & Text

All page text is directly in the HTML files — no CMS or database required. Simply open each .html file and edit the content. Key things to update on every page:

  • Page <title> tag
  • <meta name="description"> for SEO
  • Page hero title and subtitle
  • All body copy, headings, and lists
  • Pricing numbers and feature lists
  • Blog post content and dates
  • Team names, roles, and bios
  • Testimonial text and names

Services

Services are defined in plain HTML in services.html and as cards in index.html. To add, remove, or rename a service, edit the service card HTML directly. The navigation dropdown in main.js also needs updating:

// In main.js — injectHeader() — nav-dropdown section
<a href="services.html"><span>🚢</span> Your Service Name</a>
<a href="services.html"><span>✈️</span> Another Service</a>

Pricing

Update pricing on pricing.html. The billing toggle (monthly/annual) is handled by inline JavaScript at the bottom of that page — update both the display prices and the prices object:

const prices = {
  starter:  [299, 239],   // [monthly, annual]
  business: [899, 719],   // [monthly, annual]
};

Contact Information

Contact details appear in three places. Update them all:

  • main.js — SITE object (top bar + footer)
  • contact.html — Contact section HTML
  • Each page's footer — Injected from main.js

Connecting Forms

The contact form and quote form are currently client-side only. To receive submissions, choose one of these options:

Option A: Formspree (easiest)

<!-- Change the form action -->
<form action="https://formspree.io/f/YOUR_FORM_ID" method="POST">

Option B: EmailJS (no backend)

// In quote-form.js — submitQuoteForm()
emailjs.send('service_id', 'template_id', {
  from_name:  data.first_name + ' ' + data.last_name,
  from_email: data.email,
  message:    JSON.stringify(data),
});

Option C: Custom Backend (API)

// In quote-form.js — submitQuoteForm()
const response = await fetch('https://your-api.com/quotes', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(data),
});

Dark Mode

Dark mode works automatically via prefers-color-scheme media query. The toggle button in the header switches it manually and saves preference to localStorage.

To disable dark mode entirely, remove the dark-mode.css link from all HTML files and remove the initDarkMode() call in main.js.

To force dark mode by default:

<!-- Add to <html> tag -->
<html lang="en" class="dark">
