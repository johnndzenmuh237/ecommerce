Deployment Guide

How to go live — on Netlify, Vercel, GitHub Pages, cPanel, or any web server.

SwiftCargo is a static HTML website, which means deployment is straightforward — just upload the files to any web host and you're live. No database, no PHP, no Node.js required.

Netlify Recommended

🚀
Netlify — Free tier available
The easiest way to deploy. Drag and drop your folder, get a live URL in under 30 seconds. Free SSL, CDN, and custom domain support included.
netlify.com → Sites → Drag & drop folder

Option 1: Drag & Drop (fastest)

  1. Go to app.netlify.com and create a free account
  2. From the Sites dashboard, drag your entire shipping-agency-pro/ folder into the deploy drop zone
  3. Netlify deploys instantly and gives you a *.netlify.app URL
  4. Add a custom domain under Site Settings → Domain Management

Option 2: Git-connected deploy (automatic updates)

  1. Push your project to a GitHub / GitLab / Bitbucket repo
  2. In Netlify: New Site → Import from Git → Select your repo
  3. Build command: leave blank. Publish directory: ./ (or the root)
  4. Every push to main branch auto-deploys — zero configuration
# Optional: netlify.toml in project root
[build]
  publish = "."

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

Vercel

▲
Vercel — Free tier available
Lightning-fast global CDN with automatic HTTPS and Git-integrated deployments. Excellent performance analytics included.
vercel.com → New Project → Import Git Repository
  1. Push your project to GitHub
  2. Go to vercel.com → New Project → Import
  3. Select your repository. Framework Preset: Other
  4. Output directory: ./ (root). No build command needed.
  5. Deploy. You'll get a *.vercel.app URL instantly.

GitHub Pages Free

🐙
GitHub Pages — Completely free
Ideal if your code is already on GitHub. Supports custom domains and free HTTPS via Let's Encrypt.
repo → Settings → Pages → Deploy from branch
  1. Push your project to a public GitHub repository
  2. Go to repo Settings → Pages
  3. Source: Deploy from a branch → Branch: main → Folder: / (root)
  4. Your site is live at https://username.github.io/repo-name/
⚠️
Note on paths If deploying to a subdirectory (e.g. /repo-name/), update all relative paths in the HTML or use a root-relative base URL: <base href="/repo-name/"> in each <head>.

cPanel / FTP (Shared Hosting)

🖥️
cPanel / FTP — Any shared host
Works with any shared hosting provider (GoDaddy, Bluehost, SiteGround, Hostinger, etc.). Upload via cPanel File Manager or FTP client.

Via cPanel File Manager:

  1. Log into your hosting control panel
  2. Open File Manager → navigate to public_html/
  3. Upload and extract the shipping-agency-pro.zip
  4. If deploying to root: move all files from the extracted folder into public_html/ directly

Via FTP (FileZilla):

# FileZilla connection settings
Host:     ftp.yourdomain.com
Username: your_ftp_username
Password: your_ftp_password
Port:     21

# Upload to:
/public_html/           # for root domain
/public_html/subdomain/ # for subdomain/subfolder

Apache / Nginx (VPS / Dedicated)

Apache

# /etc/apache2/sites-available/swiftcargo.conf
<VirtualHost *:80>
  ServerName   yourdomain.com
  DocumentRoot /var/www/swiftcargo
  DirectoryIndex index.html

  <Directory /var/www/swiftcargo>
    AllowOverride All
    Require all granted
  </Directory>
</VirtualHost>
sudo a2ensite swiftcargo
sudo systemctl reload apache2

Nginx

# /etc/nginx/sites-available/swiftcargo
server {
  listen 80;
  server_name yourdomain.com www.yourdomain.com;
  root /var/www/swiftcargo;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  # Cache static assets
  location ~* \.(css|js|jpg|webp|png|svg|woff2)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
  }
}
sudo ln -s /etc/nginx/sites-available/swiftcargo /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

SEO Pre-Launch Checklist

🔍
Complete these before going live:
  • Update <title> on every page — unique, under 60 chars
  • Update <meta name="description"> on every page — 150–160 chars
  • Add <meta property="og:title">, og:description, og:image for social sharing
  • Create sitemap.xml and submit to Google Search Console
  • Create robots.txt in root (see below)
  • Add Google Analytics or Plausible tracking script
  • Set up Google Search Console and verify ownership
  • Ensure all images have meaningful alt attributes
  • Check page speed at PageSpeed Insights

robots.txt

User-agent: *
Allow: /
Sitemap: https://yourdomain.com/sitemap.xml

Performance Optimization

The template is already optimized, but these steps will push scores higher:

  • Compress images — Convert JPGs to WebP. Target under 150KB per image.
  • Enable Gzip/Brotli — On your server or through Cloudflare (free)
  • Use a CDN — Cloudflare free tier adds global edge caching
  • Self-host fonts — Download from Google Fonts and serve locally to avoid third-party DNS lookups
  • Lazy load images — Already applied (loading="lazy" on all non-critical images)
  • Minify CSS/JS — Use CSSNano + Terser, or a bundler like Vite

Self-hosting Google Fonts

/* 1. Download fonts from: https://fonts.google.com */
/* 2. Place in assets/fonts/ */
/* 3. Add to top of style.css */
@font-face {
  font-family: 'Barlow Condensed';
  src: url('../fonts/barlow-condensed-700.woff2') format('woff2');
  font-weight: 700;
  font-style: normal;
  font-display: swap; /* Critical for performance */
}

SSL / HTTPS

Always serve your site over HTTPS. Options:

  • Netlify / Vercel / GitHub Pages — Free SSL certificates included automatically
  • Cloudflare — Add your domain to Cloudflare free tier for instant SSL + CDN
  • Let's Encrypt (VPS) — Use Certbot to auto-provision free certificates:
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
ℹ️
HTTPS Redirect After enabling SSL, add this to your .htaccess (Apache) or Nginx config to force all HTTP traffic to HTTPS.
# .htaccess — Force HTTPS (Apache)
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
